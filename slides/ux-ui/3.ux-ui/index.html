Introducción a UX/UI

UX (User Experience) y UI (User Interface) trabajan juntos para crear productos digitales usables, accesibles y atractivos.

  • UX: Enfoque en la experiencia global del usuario.
  • UI: Diseño de la interfaz, aspecto visual y componentes interactivos.
  • Objetivo: resolver necesidades reales de usuarios mientras cumplimos objetivos de negocio.

Proceso de Diseño UX

  1. Investigación: entrevistas, encuestas y análisis de competencia.
  2. Definición: arquetipos (personas), customer journey y mapas de empatía.
  3. Ideación: brainstorming, sketches y wireframes.
  4. Prototipado: prototipos de baja y alta fidelidad.
  5. Pruebas: test de usabilidad y feedback iterativo.

Principios de Diseño de Interfaces

  • Consistencia: patrones uniformes en toda la experiencia.
  • Jerarquía Visual: uso de tamaño, color y posición para guiar al usuario.
  • Retroalimentación: respuestas claras a las acciones del usuario.
  • Accesibilidad: contraste, tipografía legible y navegación via teclado.
  • Minimizar carga cognitiva: simplificar opciones y lenguaje.

Wireframes y Prototipos

Herramientas de visualización temprana sin distracciones de estilo:

  • Wireframes: bocetos en blanco y negro para estructura y flujo.
  • Prototipos de baja fidelidad: interacciones básicas con clics.
  • Prototipos de alta fidelidad: diseño visual completo y animaciones.

Patrones de Interfaz Comunes

  • Navbar: navegación principal en la parte superior o lateral.
  • Cards: agrupación de contenido en bloques modulables.
  • Modales: ventanas emergentes para tareas puntuales.
  • Tabs: conmutación de vistas relacionadas en el mismo espacio.
  • Footer: información complementaria y enlaces secundarios.

Accesibilidad (a11y)

  • Etiquetas ARIA y roles semánticos.
  • Contraste de colores según WCAG 2.1.
  • Navegación por teclado y enfoque visible.
  • Textos alternativos en imágenes.
  • Formularios claros y mensajes de error descriptivos.

Sistemas de Diseño

  • Guías de estilo: tipografías, paletas de color y espaciado.
  • Componentes reutilizables: botones, inputs, cards, etc.
  • Tokens de diseño para mantener consistencia.
  • Documentación compartida y librerías de componentes.

Herramientas y Recursos

  • Figma, Sketch, Adobe XD para diseño y prototipado.
  • Miro, FigJam para workshops y mapas de empatía.
  • Zeplin, Storybook para handoff con desarrollo.
  • UserTesting, Lookback para pruebas de usabilidad remotas.
  • Dribbble, Behance para inspiración de diseño.
