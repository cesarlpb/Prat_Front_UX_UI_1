Introducción a React

React es una librería de JavaScript para construir interfaces de usuario de forma declarativa y basada en componentes.

  • Creada por Facebook (Meta) en 2013 y mantenida por una comunidad activa.
  • Permite descomponer la UI en componentes reutilizables y encapsulados.
  • Utiliza un Virtual DOM para actualizar la vista de forma eficiente.
  • Se programa con JSX, una sintaxis que mezcla JavaScript y marcado HTML.

Ejemplos de componentes

Componente funcional:

function Hola({ nombre }) {
  return <h1>Hola, {nombre}!</h1>;
}

Componente de clase:

class Contador extends React.Component {
  state = { count: 0 };

  render() {
    return (
      <button onClick={() => this.setState({ count: this.state.count + 1 })}>
        Has hecho clic {this.state.count} veces
      </button>
    );
  }
}

¿Qué ofrece React?

  • Componentes reutilizables y composables.
  • Sistema de props para pasar datos entre componentes.
  • Manejo de state interno para componentes interactivos.
  • Ciclo de vida de componentes (montaje, actualización y desmontaje).
  • JSX: sintaxis declarativa que facilita la escritura de UIs.
  • Virtual DOM: diffing eficiente para minimizar operaciones reales en el DOM.
  • Context API para gestión de estado global ligero.
  • Ecosistema maduro: Hooks, enrutamiento (React Router), gestión de estado (Redux, Zustand), testing, etc.

Hooks de React

1. ¿Qué son los Hooks?

  • Son funciones que permiten “enganchar” el estado y el ciclo de vida de React desde componentes funcionales.
  • Introducidos en React 16.8 para evitar el uso de clases.
  • Permiten extraer lógica de estado en funciones reutilizables (Custom Hooks).

2. Reglas de los Hooks

  1. Sólo llamarlos en el nivel superior
    No dentro de bucles, condiciones ni funciones anidadas.
  2. Sólo llamarlos desde componentes de React o Custom Hooks
    Nunca desde funciones JavaScript normales.
  3. Empezar siempre con use…
    Ejemplo: useState, useFetch, useCustomLogic.

3. useState

  • Sintaxis
    const [valor, setValor] = useState(valorInicial);
  • Retorna
    1. El estado actual (valor)
    2. Una función para actualizarlo (setValor)
  • Ejemplo 
    function Contador() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Has hecho clic {count} veces</p>
      <button onClick={() => setCount(count + 1)}>
        Incrementar
      </button>
    </div>
  );
}

4. useEffect

  • Sintaxis
    useEffect(() => {
  // efecto
  return () => {
    // limpieza (opcional)
  };
}, [dependencias]);
  • Cuándo se ejecuta
    • Al montar el componente.
    • Cuando cambian las dependencias.
    • Al desmontar (si devuelve función de limpieza).
  • Ejemplos de uso
    • Llamadas a APIs
    • Suscripciones
    • Manipulación directa del DOM

5. useContext

  • Permite consumir un Context sin <Context.Consumer>
  • Sintaxis
    const valor = useContext(MiContext);
  • Ejemplo 
    const TemaContext = React.createContext('light');

function Boton() {
  const tema = useContext(TemaContext);
  return <button className={tema}>Click me</button>;
}

6. useRef

  • Mantiene un valor mutable que persiste entre renders
  • Sintaxis
    const refElemento = useRef(null);
  • Usos
    • Referenciar nodos del DOM
    • Guardar valores sin disparar re-render

7. useReducer

  • Alternativa a useState para lógicas de estado complejas
  • Sintaxis
    const [state, dispatch] = useReducer(reducer, estadoInicial);
  • Ejemplo 
    function reducer(state, action) {
  switch (action.type) {
    case 'incrementar':
      return {count: state.count + 1};
    default:
      throw new Error();
  }
}

function Contador() {
  const [state, dispatch] = useReducer(reducer, {count: 0});
  return (
    <button onClick={() => dispatch({type: 'incrementar'})}>
      {state.count}
    </button>
  );
}

8. Hooks de rendimiento

  • useCallback(fn, deps)
    Memoriza una función para evitar recrearla en cada render.
  • useMemo(fn, deps)
    Memoriza el resultado de una función costosa.

9. Custom Hooks

  • Funciones que utilizan otros Hooks y comienzan con use
  • Permiten extraer lógica reutilizable
  • Ejemplo: Hook para fetch de datos 
    function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch(url)
      .then(res => res.json())
      .then(json => {
        setData(json);
        setLoading(false);
      });
  }, [url]);

  return { data, loading };
}

10. Buenas prácticas y recursos

  • Seguir siempre las Reglas de los Hooks.
  • Mantener los Custom Hooks enfocados en una sola responsabilidad.
  • Utilizar eslint-plugin-react-hooks para validación automática.
